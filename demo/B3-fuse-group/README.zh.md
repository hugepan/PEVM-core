# B3：融合 vs 组团

> 同源源卡的两种不同架构。
> **融合（FUS）** = 一个合成的声音（N→1）。
> **组团（GRP）** = 多个声音的编排（N→结构化合奏）。

## 对比

| 维度 | FUS（融合实体） | GRP（团卡） |
|------|:-------------:|:-----------:|
| 人格 | 单一新实体 | 多个独立个体 |
| 冲突处理 | 同一声音中的内部矛盾 | 成员间角色切换 |
| 输出风格 | 稠密、重叠的意象 | 交替发言 |
| 最适合 | 创建新的复合人格 | 模拟社会群体 |

## 超越：团卡与社会模拟

MiroFish 等项目用 LLM agent 模拟社会群体，在长时间模拟中易出现人格漂移。PEVM 的团卡解决了这个问题：

| | MiroFish | PEVM 团卡 |
|---|---|---|
| 人格 | LLM 即兴发挥 | root_loss 结构化数据 |
| 一致性 | ❌ 长时间模拟后漂移 | ✅ 稳定，底层约束控制 |
| 成本（100 agent × 100 轮） | ~$40-80 | ~$4-8（轻卡模式） |

**团卡使大规模社会模拟变得实用**——稳定的人格，10 倍低成本。
