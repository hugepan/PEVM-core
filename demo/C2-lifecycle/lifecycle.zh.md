# C3 实体生命周期 — OS 风格卡片管理

> **核心观点**：每张卡走一条主生命周期：
> `ingest → add → edit → live → delete`
>
> 派生操作（fuse/group/split/timeline）不是生命周期阶段——它们从源卡**增殖**出新实体，每张派生卡独立走自己的生命周期。不消耗源卡。


## 主生命周期

每张卡从生到死经历五个阶段：

```
ingest ──→ add ──→ edit ──→ live ──→ delete
(素材)   (生卡)   (精修)   (使用)   (终止)
```

### Step 1: ingest — 原始素材入仓

实体在成为卡之前，是散落的原始材料：传记片段、采访记录、建筑照片、论文摘要。这些原材料被收集到暂存区。

**工具**：手动放置文件。无自动化 ingest 流程——素材收集是建卡前的人工工作。

### Step 2: add — 卡片生成

从原始素材到结构化人格。这是 PEVM 最核心的转化——"从文本到人格"：

```
① 注册表写入占位人格基调（默认中性）
② LLM 读取素材 → 推断精确的认知层级 + 人格参数
③ 精确参数自动写回注册表（注册表 = 进程表）
④ 可选：精修叙事结构 + 安全机制
⑤ 可选：压缩到预算内（≤3328 token）
⑥ 自检结构完整性
⑦ 挂载原始素材到卡，供后续参考
```

**工具**：`新建卡`

**关键数据**：全过程 10-30 分钟；产出 ~3000 token 每卡；注册表自动写入，立即可被其他卡引用。

### Step 3: edit — 持续精修

卡不是一次成型。`edit` 是生命周期中最常用的操作——提升质量、更新信息、调整关系。

| 操作 | 目的 |
|------|------|
| 内容精修 | LLM 辅助重写叙事段落 |
| 预算压缩 | 削减到 token 预算内 |
| 类型升级 | 提升卡的质量等级（标准→精选→顶级） |
| 加点社交关系 | 建立跨卡连接 |
| 加张力信号 | 标注卡间对立点，使融合更有张力 |
| 调人格基调 | 直接调整认知层级或人格参数 |
| 自检 | 单卡结构健康检查 |
| 挂载资源 | 运行时附加素材/参考资料 |

### Step 4: live — 卡的使用阶段

这是卡存在的目的——在真实场景中被消费：

| 模式 | 产出 | 验证 |
|------|------|------|
| 单卡人格模拟 | 以角色身份回答任意问题 | ✅ 对照实验（受控 A/B 测试） |
| 状态修饰模拟 | 同一角色在不同状态下的表现（疲劳/低功率/分析师模式） | ✅ 多种状态修饰已验证 |
| 多卡结构化辩论 | N 角色逐轮辩论，有收敛 | ✅ 多个辩论话题 |
| 场景应用 | 实体担任专家顾问（匹配分析、技术尽调、战略咨询） | ✅ 多领域应用记录 |
| 格式渲染 | 面向不同受众的输出（技术报告、社交媒体、画廊） | ✅ 完整渲染管线已验证 |

### Step 5: delete — 实体终止

卡的生命周期终点——文件清理 + 注册表条目移除。

| 模式 | 效果 | 场景 |
|------|------|------|
| 预览（默认） | 展示将删除的内容 | 确认是本卡 |
| 永久删除 | 删文件 + 删注册表 | 完全移除 |
| 归档 | 移至冷存储 | 软删除，可手动恢复 |
| 彻底清理 | 同时清除关联的派生数据 | 完整清理 |

**设计原则**：默认预览（误删防护）；需要二次确认；派生卡独立（删融合卡不影响源卡）。


## 派生增殖（可选操作）

派生操作从一张或多张源卡生成**新实体**。每张新实体有独立的注册表条目、独立文件、独立生命周期——源卡不被消耗也不被锁定。

```
                    ┌── fuse ────→ 融合卡（N 源卡 → 1 新实体）
                    │
  源卡 ─────────────┼── group ───→ 团卡（N 源卡 → 结构化群像）
                    │
                    ├── split ────→ 子人格卡（1 源卡 → N 内在面）
                    │
                    └── timeline ──→ 时间切片卡（1 源卡 → N 人生阶段）
```

### fuse — 融合（N→1）

将 N 张源卡合并为一个新实体，生成合成人格。融合卡以单一声音说话，内含源卡间的张力。

**产出**：完整人格卡（初始反应 / 立场声明 / 核心关切 / 核心机会 / 行动倾向 / 金句）。

**人格计算**：认知层级 = 源卡算术平均；人格基调 = 加权平均（register 越高的源卡权重越大）。

**关键能力**：跨域融合——物理学家、音乐家和建筑师可以融合为一个同时拥有三种视角的实体。

**验证**：同源 3 卡分别走 fuse 和 group 路径，产出截然不同的结果——证明差异来自架构而非随机漂移。

### group — 组团（N→结构化群像）

同样的 N 张源卡，不同的架构——每张卡保留自己的声音，受团结构约束。团卡以多个声音有结构地发声。

**vs fuse**：融合 = 一个声音（合成）；组团 = 多个声音（编排）。两种方式适用于不同场景。

### split — 人格分裂（1→N）

发现同一个人的多个内在面，为每个面生成独立卡片。

**示例**：一位诗人可能分裂为"谪仙"（出世）、"求仕者"（入世）、"醉侠"（叛逆）——每张子卡有自己的认知层级、人格基调和社交关系。一位科技创始人可能分裂为"偏执创客"、"现实扭曲者"、"界面革命家"。

**关键发现**：同一段关系在不同子人格卡中的评估完全不同——系统从每个子人格的视角**重新审视**，而非模板复制。

### timeline — 时间切片（1→N·纵向）

同一个人在不同人生阶段的切片，每阶段是一张独立卡片。时间切片揭示人格如何随时间演化。

**示例**：一位创始人的旅程从大学退学 →  visionary → 被逐出 → 回归 → 巅峰；一位发明家的道路从觉醒 → 濒死体验 → 改变世界的野心。

**结构**：每张切片是完整卡片，有自己的认知层级和人格参数——数值随阶段自然变化，反映真实成长。

### 派生卡的生命周期

每张派生卡诞生后，独立走完整的生命周期：

```
fuse/split/timeline/group
    ↓
  add（注册表自动写入）
  edit（精修、压缩、加关系）
  live（上身对话、辩论、应用、渲染）
  delete（独立于源卡）
```

**关键约束**：
- 删除融合卡 → 源卡不受影响
- 删除源卡 → 派生卡不受影响（它们有自己的注册表条目）
- 同一源卡可参与多张融合、多次分裂、多个时间线


## 操作系统类比对照表

| PEVM 操作 | OS 对等物 | 说明 |
|-----------|----------|------|
| 创建卡片 | fork + exec | 从模板创建进程，分配 PID（注册表条目） |
| 精修卡片 | 热补丁 | 不重启的更新 |
| 类型升级 | renice | 提升进程优先级 |
| 挂载资源 | LD_PRELOAD | 运行时附加资源 |
| 单卡对话 | 信号 | 向进程发送消息 |
| 多卡辩论 | IPC | 多进程结构化通信 |
| 健康检查 | 系统监控 | 进程健康检查 |
| 融合 | 进程合并 | N 进程 → 1 个合并 |
| 分裂 | 受控 fork | 从 1 父进程产生 N 子进程 |
| 时间线 | ZFS 快照 | 版本化状态检查点 |
| 删除 | kill | 终止 + PID 回收 |
