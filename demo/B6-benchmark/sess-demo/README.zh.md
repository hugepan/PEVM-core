# SESS 域 · 会话代谢

> **对标**：conversation/memory 项目（Claude Memory / ChatGPT / Mem.ai / Rewind）
> **问题**：PEVM 的 SESS 卡跟传统记忆系统有什么本质不同？
> **答案**：它们在存聊天记录，我们在存认知增量。

## 背景

大部分"AI 记忆"方案做的事：把对话全文存下来，下次需要时整段塞回上下文。问题是——几百轮对话塞进去，token 爆炸，信号淹没在噪声里。

SESS 域的做法：每次会话结束后，自动提炼为一张结构化卡片——只保留认知骨架，舍弃对话中的冗余和噪声。单卡 ~3000t，精准加载。

## 验证材料

**最强证据**：[fusion-insight.zh.md](fusion-insight.zh.md) — 三段独立的工程经历（bug根因追溯、跨项目复用、自举验证）被关联后，回答了一个问题："一个项目团队如何实现自我进化？"

结果中出现了"1000个bug背后只有3个认知裂缝"、"把一个人的直觉变成所有人的直觉储备库"、"把救火队变成消防检查员"等洞见——这些不是从某一次对话中提取的，是三次独立对话被关联后涌现的新认知。传统 memory 项目做不到。
