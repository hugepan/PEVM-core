# A3：状态修饰

> **问题**：人格参数（root_loss）是因果的还是装饰的？
> **答案**：有因果关联的。改数值，输出产生定向变化——可观察且可复现。

## 实验

同一角色（李白）、同一 LLM（ds_flash）、同一问题——仅改变状态：

| 状态/模式 | 作用 | 对输出的影响 |
|-----------|------|-------------|
| 重感冒 | 语义型状态修饰 | 更简短、低能量，但仍不离角色 |
| 20% | 算法型状态修饰 | 创造性展开减少，回答更直接 |
| Consultant | 角色模式修饰 — `--role consultant` | 从第一人称人格切换到第三人称分析；同一卡、同一问题，两种模式并排对比 |

## 超越实验

状态修饰不只是技术 demo。它指向更大的可能：**一张能反映实时状态的人格卡**——工作模式、休息模式、专注模式、空闲模式。一张知道"我今天累了"并相应调整的卡。

结合社交协议，这就变成了**一种新型的 profile 卡片**：不是一个静态简介，而是一个反映用户当前状态的活跃存在。状态修饰 = 社交协议的基础设施。
